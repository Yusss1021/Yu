## 5.3 局限性与合规分析

### 5.3.1 系统局限性

1. 规则库规模仍有限，主要用于验证方法链路；  
2. 服务版本识别精度依赖目标响应质量；  
3. 资产画像目前以人工维护为主，自动化资产分级尚未实现；  
4. 风险模型未纳入攻击路径、网络拓扑纵深等复杂因子。

### 5.3.2 合规与伦理边界

1. 仅在授权资产与授权网段执行扫描；  
2. 禁止对未授权公网/第三方主机发起探测；  
3. 实验漏洞结果仅用于教学与研究，不用于攻击用途；  
4. 报告与数据库中如含敏感信息，应做脱敏存储与最小化展示。

### 5.3.3 工程风险控制

1. 参数严格校验（端口、方法、网段）避免误操作；  
2. 对比前校验扫描ID有效性，避免错误分析；  
3. SQLite 启用 `WAL + busy_timeout` 提升并发稳定性。

## 5.4 可复现实验步骤（含截图清单）

### 5.4.1 复现实验命令

说明：端口与服务识别由 `nmap -sV` 阶段完成，`syn` 用于主机存活补充（需 root 或 `CAP_NET_RAW`）；若权限不足，可仅使用 `icmp`。

补充说明：当环境未安装 nmap 或服务版本无法被识别时，报告将出现“需要手动确认漏洞”的提示。其含义为：规则命中仅在服务/产品层面成立，但由于版本字段缺失且 `version_rule` 非通配（非 `*`），系统无法自动验证版本条件，因而将该条结果标注为低置信度（LOW），以引导复现与审阅时进行人工核对。

1) 环境准备

```bash
cd /home/yu/intra_vuln_assessor
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

2) 启动本地靶场

```bash
./lab/start_demo_lab.sh
```

3) 扫描（无资产画像）

```bash
python3 main.py scan --target 127.0.0.1/32 --methods icmp,syn --ports 2222,6379 --name exp_no_profile
```

4) 扫描（加载资产画像）

```bash
python3 main.py scan --target 127.0.0.1/32 --methods icmp,syn --ports 2222,6379 --asset-profile docs/asset_profile.example.json --name exp_with_profile
```

5) 对比两次扫描

```bash
python3 main.py compare --base <ID_A> --new <ID_B>
```

6) 启动Web并提交并发任务

```bash
python3 main.py web --host 127.0.0.1 --port 5000 --max-concurrent 3
```

7) 停止靶场

```bash
./lab/stop_demo_lab.sh
```
